## 🎯 解决的问题

### ❌ 之前的问题（v2.0.0 修复）
1. 批量生成章节是顺序执行，N 章需要 N × 60s，非常缓慢
2. 进度条只能基于时间估算，无法展示真实进度
3. 定时调度也是顺序执行多章生成
4. 用户无法实时了解生成到了哪个阶段

### ✅ 现在的改进
1. **并发批量生成**: N 章耗时 ≈ 单章耗时（~60s），速度提升 3-5x
2. **真实进度上报**: 后端分阶段上报进度（10%→15%→30%→80%→90%→100%）
3. **智能定时调度**: 使用 `asyncio.gather()` 并发执行定时生成
4. **轮询加速**: 从 2s 提升至 1.5s，进度响应更快

### 性能提升对照

| 场景 | 优化前 | 优化后 | 提升倍数 |
|------|--------|--------|:--------:|
| 批量生成 3 章 | ~180s | ~60s | **3x** |
| 批量生成 5 章 | ~300s | ~60s | **5x** |
| 进度精度 | 估算 | 真实 | — |
| 轮询间隔 | 2s | 1.5s | — |
